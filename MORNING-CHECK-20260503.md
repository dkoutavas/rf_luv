### Quick status (run first)

```bash
# Total messages + rate + unique flights/tails in the 21h soak window
docker exec clickhouse-acars clickhouse-client \
  --user acars --password acars_local --database acars --query "
SELECT
  count()                                                   AS total_messages,
  round(count() / 21, 0)                                    AS msgs_per_hour,
```

### Message rate by hour (trend)

```bash

  --user acars --password acars_local --database acars --query "

  toStartOfHour(timestamp)  AS hour_utc,
  count()                   AS messages,
  countDistinct(flight)     AS flights
```

Expect quiet overnight (00:00–05:00 UTC), picking up at 05:00+ as LGAV morning

```bash

WHERE tail != '' AND timestamp > now() - INTERVAL 12 HOUR

wc -l /tmp/acars_tails.txt   # expect 10–50 distinct tails over 12h at LGAV
```

**Step 2 — extract ADS-B callsigns/hex (from ADS-B ClickHouse, Windows host):**

```sql
-- Run against adsb database on the Windows host (port 8123 / localhost or wherever)
SELECT DISTINCT callsign
FROM aircraft_latest
WHERE timestamp > now() - INTERVAL 12 HOUR
  AND callsign != ''
ORDER BY callsign
```

Note: ADS-B stores `callsign` (flight number like `AEE123`), not `tail`
(registration like `SX-DGT`). ACARS carries both. For overlap analysis:
